PowerShell Solution: Script not digitally signed

2021-02-11
“.ps1 is not digitally signed. The script will not execute on the system.” This error is clearly not really a typical error, but can be a blocker, nonetheless. It’s more of a security feature than an error. Simple solution if you’re running this script from another service, or as an Azure Runbook, before executing the Invoke-Command: Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass This will allow the powershell session process itself to bypass the Exec policy, and after termination the next powershell process has the default user or system policy.
